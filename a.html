Core Principles
Three architectural hypotheses validated at runtime
01

Modular Architecture Reduces Orchestration Overhead

Subsystems are decomposed and independently schedulable. The LayerStack dispatcher routes execution without coupling.

02

Batch Rendering Minimizes Draw-Call Submission Cost

Geometry accumulated on CPU, flushed in a single GPU submission. Dramatic reduction in API call overhead.

03

ECS Improves Scalability Through Contiguous Iteration

Sparse-set storage ensures cache-friendly traversal. Components iterated linearly, systems decoupled from entities.

Why Separation Matters
Editor operations run in their own application layer. They cannot interfere with runtime frame scheduling. The engine sees the editor as just another consumer of the FrameBuffer output — identical to how the game would use it.
